# Meteo

A Laravel 11 application that fetches daily weather forecast data from multiple providers (Open-Meteo and WeatherAPI.com), stores hourly and daily forecasts in MySQL, and runs automatically via the Laravel Scheduler.

---

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) 24+
- [Docker Compose](https://docs.docker.com/compose/install/) v2+

No local PHP or Composer installation is required — everything runs inside Docker.

---

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/cgeorgoudi/meteo-weather-service.git meteo
cd meteo
```

### 2. Configure environment

```bash
cp .env.example .env
```

Open `.env` and set your WeatherAPI.com key:

```dotenv
WEATHER_API_KEY=your_actual_key_here
```

You must also set `APP_KEY`. Generate one with:

```bash
# If you have PHP locally:
php -r "echo 'base64:'.base64_encode(random_bytes(32)).PHP_EOL;"

# Or use an online generator and paste the result as:
APP_KEY=base64:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Optionally, override the default seeded location:

```dotenv
DEFAULT_LOCATION_NAME=London
DEFAULT_LOCATION_LAT=51.5074
DEFAULT_LOCATION_LON=-0.1278
DEFAULT_LOCATION_TIMEZONE=Europe/London
```

### 3. Build and start

```bash
docker compose up -d --build
```

On first start the `app` container will:
1. Wait for MySQL to be healthy.
2. Run `php artisan migrate --force` to create all tables.
3. Run `php artisan db:seed --force` to seed the default location and provider records.
4. Start `php artisan schedule:work` which runs the scheduler every minute.

The scheduler is configured to call `weather:fetch` daily at midnight UTC.

## Manual fetch

Run a fetch immediately without waiting for the scheduler:

```bash
docker compose exec app php artisan weather:fetch
```

You should see output like:

```
Starting weather data fetch...
[2026-03-22 12:00:00] Fetching weather for [Athens] from [Open-Meteo]
```

## How to add a new location

To add a new city to the fetcher, you can insert it directly into the database.

### Using Docker & SQL (Recommended)

Run this command from your terminal to add Berlin, for example:

```bash
docker compose exec mysql mysql -u meteo -psecret -e "INSERT INTO locations (name, latitude, longitude, timezone, is_active, created_at, updated_at) VALUES ('Berlin', 52.5200, 13.4050, 'Europe/Berlin', 1, NOW(), NOW());" meteo
```

## How to add a new weather provider

1. **Create the provider class** implementing `App\Weather\Contracts\WeatherProviderInterface`:

```php
<?php

namespace App\Weather\Providers;

use App\Models\Location;
use App\Weather\Contracts\WeatherProviderInterface;
use App\Weather\Data\DailyEntry;
use App\Weather\Data\HourlyEntry;
use App\Weather\Data\WeatherData;
use Carbon\Carbon;
use GuzzleHttp\Client;

class MyNewProvider implements WeatherProviderInterface
{
    public function __construct(private readonly Client $client) {}

    public function getName(): string
    {
        return 'My New Provider';
    }

    public function fetch(Location $location, int $days): WeatherData
    {
        // Call the API, parse the response, return a WeatherData instance
        // containing arrays of DailyEntry and HourlyEntry objects.
        return new WeatherData(daily: [], hourly: []);
    }
}
```

2. **Register the binding** in `app/Providers/AppServiceProvider.php`:

```php
$this->app->bind(\App\Weather\Providers\MyNewProvider::class, function () {
    return new \App\Weather\Providers\MyNewProvider(new \GuzzleHttp\Client());
});
```

3. **Add the class** to `config/weather.php`:

```php
'providers' => [
    \App\Weather\Providers\OpenMeteoProvider::class,
    \App\Weather\Providers\WeatherApiProvider::class,
    \App\Weather\Providers\MyNewProvider::class,  // <-- add here
],
```

4. **Re-seed** to insert the new provider record into the database:

```bash
docker compose exec app php artisan db:seed --class=WeatherProviderSeeder --force
```

The next `weather:fetch` run will automatically include the new provider for every active location.

## Database schema

### `locations`

| Column     | Type            | Description                          |
|------------|-----------------|--------------------------------------|
| id         | bigint (PK)     |                                      |
| name       | varchar         | Human-readable name (e.g. "Athens")  |
| latitude   | decimal(10,7)   |                                      |
| longitude  | decimal(10,7)   |                                      |
| timezone   | varchar         | IANA timezone string                 |
| is_active  | tinyint(1)      | Set to 0 to exclude from fetches     |
| created_at | timestamp       |                                      |
| updated_at | timestamp       |                                      |

### `weather_providers`

| Column     | Type        | Description                                      |
|------------|-------------|--------------------------------------------------|
| id         | bigint (PK) |                                                  |
| name       | varchar     | Human-readable name (e.g. "Open-Meteo")          |
| class      | varchar     | Fully-qualified class name of the provider       |
| is_active  | tinyint(1)  | Set to 0 to disable without removing             |
| created_at | timestamp   |                                                  |
| updated_at | timestamp   |                                                  |

### `daily_forecasts`

| Column              | Type            | Description                            |
|---------------------|-----------------|----------------------------------------|
| id                  | bigint (PK)     |                                        |
| location_id         | bigint (FK)     |                                        |
| weather_provider_id | bigint (FK)     |                                        |
| forecast_date       | date            |                                        |
| temp_min            | decimal(5,2)    | Celsius                                |
| temp_max            | decimal(5,2)    | Celsius                                |
| precipitation_sum   | decimal(7,2)    | mm                                     |
| fetched_at          | timestamp       | When this row was last fetched         |
| created_at          | timestamp       |                                        |
| updated_at          | timestamp       |                                        |

Unique index on `(location_id, weather_provider_id, forecast_date)` — rows are upserted on each fetch.

### `hourly_forecasts`

| Column              | Type            | Description                            |
|---------------------|-----------------|----------------------------------------|
| id                  | bigint (PK)     |                                        |
| location_id         | bigint (FK)     |                                        |
| weather_provider_id | bigint (FK)     |                                        |
| forecast_datetime   | timestamp       |                                        |
| temperature         | decimal(5,2)    | Celsius                                |
| precipitation       | decimal(7,2)    | mm                                     |
| fetched_at          | timestamp       | When this row was last fetched         |
| created_at          | timestamp       |                                        |
| updated_at          | timestamp       |                                        |

Unique index on `(location_id, weather_provider_id, forecast_datetime)` — rows are upserted on each fetch.

## Scalability notes

- **Multiple locations**: Insert rows into the `locations` table. The fetcher iterates all active locations automatically — no code changes required.
- **Multiple providers**: Implement `WeatherProviderInterface`, register in the service container and in `config/weather.php`. No changes to `WeatherFetcher` are needed.
- **Provider toggling**: Set `is_active = 0` on any `weather_providers` row to stop fetching from that source without touching code or config.
- **Forecast horizon**: Change `forecast_days` in `config/weather.php` to increase or decrease how many days ahead are fetched. WeatherAPI.com is capped at 14 days by its free tier; Open-Meteo supports up to 16 days.
- **Fetch frequency**: Modify the scheduler expression in `bootstrap/app.php`. For example, `->everyThreeHours()` instead of `->dailyAt('00:00')` to refresh forecasts more often.
